# Documentación de NexTask

Esta carpeta concentra la documentación oficial del proyecto. Todo el equipo debe consultarla antes de modificar arquitectura, agregar módulos o tocar infraestructura.

## Secciones

```txt
Documentacion/
  README.md
  frontend/          Guías del frontend, UI, componentes, páginas y estilos
  backend/           Guías del backend, server, APIs, base de datos y servicios
  infraestructura/   Docker, PostgreSQL, variables de entorno y despliegue futuro
  arquitectura/      Decisiones arquitectónicas y estructura del proyecto
  equipo/            Flujo Git, convenciones y trabajo colaborativo
  Base/              Documentación heredada inicial
```

## Principios del proyecto

- Base profesional antes que lógica compleja.
- Separación clara de responsabilidades.
- Código legible y mantenible.
- TypeScript estricto.
- UI moderna, responsive y accesible.
- Documentación obligatoria para decisiones importantes.
- Preparado para crecer hacia autenticación, tickets, RAG, PostgreSQL, Prisma, Redis, APIs y WebSockets.
